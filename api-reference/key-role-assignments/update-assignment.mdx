---
title: 'Update Role Assignment'
api: 'PUT /api/v1/key-role-assignments/{assignmentId}'
description: 'Update a specific internal team assignment'
---

## Path Parameters

<ParamField path="assignmentId" type="string" required>
  The unique ID of the role assignment to update
</ParamField>

## Request Body

<ParamField body="userEmail" type="string">
  Updated internal team member email address
</ParamField>

<ParamField body="roleKey" type="string">
  Updated role key
</ParamField>

<ParamField body="tenure" type="object">
  Updated tenure period
  
  <Expandable title="Tenure Object">
    <ParamField body="startDate" type="string">
      Updated start date in YYYY-MM-DD format
    </ParamField>
    
    <ParamField body="endDate" type="string">
      Updated end date in YYYY-MM-DD format (set to end an assignment)
    </ParamField>
  </Expandable>
</ParamField>

## Example Requests

### End a CSM Assignment (Team Member Transition)
```bash
curl -X PUT https://api.arali.ai/api/v1/key-role-assignments/{assignmentId} \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "tenure": {
      "endDate": "2024-06-30"
    }
  }'
```

### Change Assigned Team Member
```bash
curl -X PUT https://api.arali.ai/api/v1/key-role-assignments/{assignmentId} \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "userEmail": "new.csm@yourcompany.com",
    "tenure": {
      "startDate": "2024-07-01"
    }
  }'
```

## Response

Returns the updated role assignment record.

## Notes

Common use cases:
- Set `endDate` when a team member transitions off an account
- Update `userEmail` when reassigning to a different team member
- Maintain historical records by setting end dates rather than deleting assignments
